# 🤖 CUBELINK Studio v2.6.4 → v2.6.5 작업 이어가기

## 📌 프로젝트 개요
- 이름: **CUBELINK Studio** (아두이노 4축 로봇팔 교육용 블록코딩 IDE)
- 현재 버전: **v2.6.4** (GitHub 업로드 완료)
- 다음 버전: **v2.6.5** (각도/방향 정정 + 잔여 이슈 정리)
- 파일 구조: 단일 `index.html` (HTML+CSS+JS 통합), `models/` 폴더에 GLB 파일

## 🛠️ 사용 기술
- Blockly (libs/blockly/blockly.min.js + ko.js — 로컬)
- Three.js r128 (CDN) + GLTFLoader + OrbitControls
- 3D 모델: base.glb / lower.glb / upper.glb / grip01.glb / grip02.glb

## 🐛 v2.6.5에서 정정할 이슈 (우선순위 순)

### 1. PIN 11 열림/닫힘 방향 반전
- 현재: `const openRad = radian((angle - 90) * -0.33);`
- 0°/180°가 실제 로봇팔과 반대로 동작 → 부호만 뒤집기 (또는 0.33으로)
- 실제 하드웨어 매핑 확정 필요

### 2. PIN 6 부호 통일
- 현재: `window.baseGroup.rotation.y = -radian(angle - 90);`
- 다른 핀과 일관성 위해 마이너스 제거 검토

### 3. HTML 서보 초기값 불일치
- `servoAngle10`: 67 → 90 으로 변경
- `servoAngle11`: 45 → 90 으로 변경
- (모든 서보 90°가 기준 자세)

### 4. servoFill 바 동기화 누락
- `Sim.setServo()` 안에서 `servoFill{pin}` width와 `servoAngle{pin}` 텍스트 자동 갱신 코드 추가

### 5. progressBadge 초기 텍스트
- `<span id="progressBadge">진행률: 0 / 7</span>` → `0 / 9`

### 6. PIN 9, 10 클램프 범위 검토
- 현재 PIN 9: 30~150, PIN 10: 0~135 (임의 제한)
- 실제 하드웨어 한계와 일치하는지 확인 후 0~180 통일 검토

### 7. 실제 하드웨어 각도 매핑 확정 (가장 중요)
- 각 서보의 실제 설치 방향과 0°/90°/180° 동작 매핑이 미확정
- PIN 6: Y축, 0°=??, 90°=정면, 180°=??
- PIN 9: 회전축=?, 각도별 자세=?
- PIN 10: 회전축=?, 각도별 자세=?
- PIN 11: Y축(서보가 위→아래 방향으로 grip01에 연결), 각도별=?

### 8. BLOCKOLI 잔여 코드 정리 (선택)
- `cubelink_v2_*` 블록 정의/생성기/시뮬레이터 코드 제거 여부 결정

## 💡 작업 스타일 (이전 대화에서 정착된 방식)
- 코드는 **콘솔(F12)에서 먼저 실시간 테스트** → 만족 시 영구 반영
- 한 번에 한 이슈씩 (우선순위대로)
- 회전 방향 모를 땐 X/Y/Z 축 순서로 ±0.5씩 테스트
- 위치 어긋남 → Box3로 size/center 진단 후 오프셋 계산
- 사용자가 GLB 파일을 직접 편집 가능 (이미 grip02.glb 재생성한 경험 있음)
- 짧고 명확한 응답 선호, 표/체크리스트 활용

- 코드는 정상 동작 (콘솔 에러 없음)
